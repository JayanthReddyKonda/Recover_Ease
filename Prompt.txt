# WHAT THIS APP IS
================================================================

A post-discharge patient recovery monitoring system:
- Patients log daily symptoms (voice/text/manual sliders)
- Doctors monitor via intelligent triage dashboard
- Hybrid AI escalation engine (7 rules + Groq LLM) alerts doctors
- Real-time Socket.IO alerts
- Patient accepts/rejects doctor connection requests
- Resend email alerts (SOS + caregiver notifications)
- Production-grade UI with Framer Motion animations

# BACKEND API — 23 ENDPOINTS (ALL WORKING)
================================================================

All endpoints return { success, data, message } wrapper.
Base URL: http://localhost:8000 (or ngrok public URL)
Auth: Bearer token in Authorization header

--- Auth (4 endpoints) ---
POST /api/auth/register        Body: { email, password, name, role: PATIENT|DOCTOR }
POST /api/auth/login           Body: { email, password } -> { token, user }
GET  /api/auth/me              -> current user profile
PATCH /api/auth/profile        Body: { name?, surgery_date?, surgery_type?, caregiver_email? }

--- Symptoms (5 endpoints) ---
POST /api/symptoms             Body: { pain_level?, fatigue_level?, mood?, sleep_hours?, appetite?, energy?, temperature?, notes? }
                               -> Auto: AI enrichment -> escalation check -> milestones -> socket notify
GET  /api/symptoms             Query: ?limit=&offset= -> paginated symptom logs
GET  /api/symptoms/today       -> today's log or null
GET  /api/symptoms/summary     -> { avg_pain, avg_mood, avg_energy, avg_sleep, total_logs, streak }
GET  /api/symptoms/trend       Query: ?days=14 -> daily data points

--- Requests (7 endpoints) ---
POST   /api/requests              Body: { to_email } -> send connection request
GET    /api/requests/pending      -> list pending incoming requests
POST   /api/requests/{id}/accept  -> accept + link doctor-patient
POST   /api/requests/{id}/reject  -> reject request
GET    /api/requests/my-doctor    -> patient's assigned doctor
GET    /api/requests/my-patients  -> doctor's patient list
DELETE /api/requests/{id}/disconnect -> remove doctor-patient link

--- Patients (4 endpoints) ---
GET   /api/patients/me/profile             -> full profile + stats + milestones + recovery stage
GET   /api/patients/{patient_id}/full      -> doctor's full view (logs, escalations, milestones, AI)
POST  /api/patients/sos                    -> CRITICAL escalation + email + socket alert
PATCH /api/patients/escalations/{id}       Body: { status: ACKNOWLEDGED|RESOLVED, notes? }

--- AI (2 endpoints) ---
GET /api/ai/insight                -> AI patient insight (summary, tips, encouragement, warning_signs)
GET /api/ai/summary/{patient_id}   -> AI clinical summary for doctor (overview, trends, risk_factors, recommendations)

--- Health (1 endpoint) ---
GET /health -> { status, environment, uptime }

================================================================
# DATABASE SCHEMA (5 tables)
================================================================

users:
  id (UUID PK), email (unique), password_hash, name,
  role (PATIENT|DOCTOR), surgery_date, surgery_type,
  caregiver_email, doctor_id (FK self), created_at, updated_at

doctor_patient_requests:
  id (UUID PK), from_id (FK users), to_id (FK users),
  status (PENDING|ACCEPTED|REJECTED), created_at
  Unique: (from_id, to_id)

symptom_logs:
  id (UUID PK), patient_id (FK users), date,
  pain_level (1-10), fatigue_level (1-10), mood (1-10),
  sleep_hours (0-24), appetite (1-10), energy (1-10),
  temperature, notes, parsed_symptoms (JSON), ai_insight (JSON),
  created_at

escalations:
  id (UUID PK), patient_id (FK users), symptom_log_id (FK),
  doctor_id (FK users), severity (LOW|MEDIUM|HIGH|CRITICAL),
  status (OPEN|ACKNOWLEDGED|RESOLVED), rule_results (JSON),
  ai_verdict (JSON), is_sos (bool), doctor_notes, created_at,
  resolved_at

milestones:
  id (UUID PK), patient_id (FK users), milestone_key,
  title, icon, earned_at
  Unique: (patient_id, milestone_key)

================================================================
# SOCKET.IO EVENTS (Backend emits these)
================================================================

Server -> Client Events:
  'escalation_alert'     -> { patient_id, severity, is_sos, message }
  'sos_alert'            -> { patient_id, patient_name, message }
  'connection_request'   -> { request_id, from_user }
  'request_accepted'     -> { request_id }
  'request_rejected'     -> { request_id }

Client -> Server Events:
  'join_doctor_room'     -> doctor_id (joins doctor:{id} + doctor_alerts rooms)
  'join_patient_room'    -> patient_id (joins patient:{id} room)
